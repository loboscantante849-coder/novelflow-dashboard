# NovelFlow 推荐官经济体系 V2 — 保守落地版

> 设计原则：**干活的吃肉，不干活的喝汤，不干活的留不住。**
> 与V1（代币+博弈版）的关系：V2是V1的可落地子集，核心飞轮一致，去掉法律红线。

---

## 一、核心飞轮

```
作者付费买曝光 → 平台收过路费 → 分发奖金/积分刺激推荐官 → 推荐官推广带来读者 → 读者数据反哺 → 作者愿意付更多
```

三层角色，各取所需：

| 角色 | 想要什么 | 付出什么 | 得到什么 |
|------|---------|---------|---------|
| 作者 | 曝光 → 读者 → 订阅收入 | 推广费（美元） | 推荐位 + 数据报告 |
| 推荐官 | 赚钱 + 成就感 | 推广劳动（发帖/视频/拉新） | 佣金 + 积分 + 勋章权益 |
| 读者 | 好书 | 注意力 | 免费章节 + 推荐匹配 |

## 二、积分体系（NF Points）

### 2.1 基本定义

- **NF Points 是平台积分，不是货币，不可提现**
- 用途：兑换权益、参与打榜、解锁功能
- 获取方式：干活赚，不能买（与美元不锚定，避免Money Transmitter风险）

### 2.2 获取规则

| 行为 | 积分奖励 | 频率限制 |
|------|---------|---------|
| 推荐一本书（成功创建链接） | +50 pts | 无限 |
| 链接被点击（visit） | +2 pts/次 | 每链接每日上限100pts |
| 链接产生激活（activate） | +20 pts/次 | 无限 |
| 链接产生新增用户 | +50 pts/次 | 无限 |
| 创建AI视频reel | +30 pts/条 | 每日7条上限 |
| 拉新注册 | +100 pts/人 | 无限 |
| 连续7天活跃 | +200 pts bonus | 每周1次 |
| 社媒帖子被平台精选 | +150 pts | 不定期 |

### 2.3 消耗规则

| 行为 | 积分消耗 | 说明 |
|------|---------|------|
| 推荐官打榜投票 | 10 pts/票 | 为某本书投票冲榜 |
| 解锁先行者勋章 | 2000 pts | 一次性，永久有效 |
| 兑换额外视频额度 | 100 pts/条 | 每日最多+3条 |
| 兑换推荐位加成 | 500 pts | 24h内推荐链接优先展示 |
| 兑换提现手续费减免 | 300 pts/次 | 单次提现免手续费 |

### 2.4 设计逻辑

- 积分**只进不出**，不能换成美元 → 不是货币，不需要牌照
- 积分**只能换权益** → 驱动用户留在生态内继续干活
- 积分获取和真实推广效果绑定 → 没有泡沫，每个积分背后都是真实数据

## 三、作者推广付费（Boost）

### 3.1 推广套餐

| 套餐 | 价格 | 推荐官分成 | 平台留存 | 奖池注入 |
|------|------|-----------|---------|---------|
| Bronze | $50 | $25 (50%) | $20 (40%) | $5 (10%) |
| Silver | $100 | $55 (55%) | $35 (35%) | $10 (10%) |
| Gold | $300 | $180 (60%) | $90 (30%) | $30 (10%) |

### 3.2 分成逻辑

- **推荐官分成**：按推广效果（激活+新增）分配给推广该书的推荐官
- **平台留存**：覆盖服务器、API、视频生成成本
- **奖池注入**：进入周榜奖金池

### 3.3 推广效果

作者付费后，该书获得：
- 推荐排行榜加权（付费书排名靠前）
- 推荐官分成比例提升（吸引更多推荐官推）
- 周榜展示位优先

## 四、推荐官打榜系统

> 这是博弈玩法的保守替代。不是投注，是免费投票。

### 4.1 机制

- 每周一个推荐榜单，展示Top 20书籍
- 推荐官用手中的**积分**为某本书投票
- 投票不花真钱，花的是劳动换来的积分
- 每票10 pts，同一本书每人每周最多投20票

### 4.2 榜单权重

```
最终得分 = 作者推广付费权重(40%) + 推荐官投票权重(30%) + 自然数据权重(30%)
```

- 付费权重：作者付了推广费，直接加权
- 投票权重：推荐官用积分投票，反映社区热度
- 数据权重：真实阅读量、激活率

### 4.3 榜单奖励

| 排名 | 奖金 | 积分奖励 |
|------|------|---------|
| #1 | $30 | +500 pts |
| #2 | $20 | +300 pts |
| #3 | $15 | +200 pts |
| #4-5 | $10 | +100 pts |
| #6-10 | $5 | +50 pts |
| #11-20 | — | +20 pts |

- 奖金从**奖池**发放（来源：作者推广费的10%）
- 奖金是真美元，可提现
- 如果当周奖池不足，按比例缩减

### 4.4 投票推荐官奖励

给该书投票且该书上榜的推荐官：
- 该书排名越高，投票推荐官获得额外积分奖励
- 鼓励推荐官研究书的质量，而不是随便投

## 五、先行者勋章

### 5.1 获取条件

满足以下**任意一个**即可解锁（+消耗2000积分）：

| 条件 | 门槛 |
|------|------|
| 累计推广激活 | ≥ 100人 |
| 累计拉新注册 | ≥ 30人 |
| 累计创建reels | ≥ 20条 |
| 累计推广收入 | ≥ $50 |

### 5.2 勋章权益

| 权益 | 普通推荐官 | 先行者 |
|------|-----------|--------|
| 每日reels额度 | 7条 | 20条 |
| 提现手续费 | 平台扣5% | 平台包了（0%） |
| 佣金比例 | 50%起 | 55%起 |
| 空投/任务 | 普通奖励 | 1.5x奖励 |
| 推荐位加成 | 无 | 每周1次免费24h加成 |
| 指定区书籍 | 标准分成 | 额外+5%推广权益 |

### 5.3 设计逻辑

- 勋章是**劳动成果**的证明，不是买的 → 没有二级市场
- 勋章效用绑定真实收益 → 有动力去获取
- 高额度 + 零手续费 = 干得越多省得越多 → 留住核心贡献者

## 八、经济模型 — 可持续性验证

### 8.1 资金流向

```
流入（美元）
├── 作者推广费（主力）
├── 广告收入（补充）
└── 订阅收入（补充）

流出（美元）
├── 推荐官佣金（最大头）
├── 周榜奖金（奖池10%）
├── 服务器/API成本
└── 提现手续费损失（先行者0%）

内部循环（不流出）
├── 积分获取 → 积分消耗（留在生态）
├── 勋章解锁 → 勋章权益（驱动更多劳动）
└── 打榜投票 → 积分回收
```

### 8.2 盈亏平衡点

假设：
- 10个作者购买Silver套餐 = $1,000/月
- 推荐官分成55% = $550流出
- 奖池10% = $100流出
- 平台毛收入 = $350

如果：
- 20个活跃推荐官，平均月收入$27.5
- 服务器+API成本 ≈ $100/月
- **平台净收入 ≈ $250/月**

### 8.3 泡沫控制

| 泡沫来源 | 控制手段 |
|---------|---------|
| 积分通胀 | 消耗场景（打榜/勋章/加成）持续回收 |
| 周榜奖金 | 上限封顶（奖池不足按比例缩减） |
| 提现压力 | 7天冷静期 + 最低$5门槛 |
| 刷单 | 激活/新增需归因验证，7天冷静期可回溯 |

## 九、与V1（代币版）的映射

| V1 代币版 | V2 保守版 | 差异 |
|-----------|----------|------|
| 代币（锚定$1） | 积分（不锚定） | 积分不可提现，避Money Transmitter |
| 投注阅读量 | 打榜投票 | 花积分不花钱，避赌博 |
| 代币合成勋章 | 积分兑换勋章 | 逻辑相同，介质不同 |
| 代币兑换分红 | 阶梯佣金加成 | 按效果分，不按持有量分 |
| 奖池瓜分2% | 周榜奖金（10%奖池） | 奖金有真美元来源 |
| 空投 | 任务奖励 | 奖励绑定劳动，不凭空发 |
| 二级市场 | 无 | 完全去掉，避证券风险 |

V2的每一个机制都在V1中有对应，只是把"钱"换成"积分"，把"投注"换成"投票"。等合规到位后，可以把积分升级为代币、把投票升级为博弈，**不需要重构架构**。
